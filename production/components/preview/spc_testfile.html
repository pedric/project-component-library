Spacecraft is a tool for building websites in which you are in control- of and can work based on components. When working with this method, wouldn't it be quite handy to be able to reuse and develop these components between projects? I am now working with a project with the aim to find- and implement a way to distribute and versioncontrol components.

To use Spacecraft together with the ready-to-use components, start with cloning this repo: https://github.com/pedric/spacecraft-starterkit, which is a version of Spacecraft-Starter in where another customised version of Spacecraft lives. This kit are expanded with the tasks for importing components direct to your project with a few commands.

Quick start guide

Clone/download the repo above, then...

npm install
...and to get this actual component downloaded run...
npm install --save spc_testfile
...followed by this to get the files into your project...
gulp import --component spc_testfile
...and to start up Spacecraft you just use...
gulp
The last step is to import the scss, in... 
./src/scss/components/_components.main.scss
...add... 
@import "views/spc_testfile/spc_testfile";

spc_testfile

There is no limitation to what a component is, this for example, could easily being a template beased on different smaller components but as is now it is a component which contains an article. If wrapping this in a head-, header- and footer component would be an example of a complete page-template based on four components. But in first hand the spc_testfile is to test and develop the structure, conventions, distribution etc for a component. If you look at this repo or open the folder /spacecraft_starterkit /node_modules /spc_testfile after the quick start guide you will find a structure which is a suggestion of how to structure components. This match well with how the different filetypes are handled in Spacecraft after importing them.

  • index.js
  • package.json
  • docs/
  • icons/
  • img/
  • js/
  • view/
  • changelog.md
  • readme.md
  • license.txt

The copy/paste

Somewhere you will do the "copy/paste", sure you can be developing your component directly and push it to NPM/GIT, but probably you will enhance components while working on them in different projects. In the structure as it is built now you will have to do this "copy/paste" since the import process is copying components into a project and from there one are not working with the original version.

Using this component

Follow the quick start guide and then customize it as you feel fits your project, this content is imported from the spc_testfile.config.json file in the ./view folder.
